关于木犀后台管理平台的一些想法

github webhook

                /-> webhook -> (docker-compose群) -> 服务器1 -> nginx监控
应用(git代码托管)- ->  ....           ....              ....
                \-> webhook -> (docker-compose群) -> 服务器2 -> nginx监控

以github webhook为纽带, 就可以把应用和部署应用的服务器结合, 并在此基础上开展应用、服务器、容器监控.
