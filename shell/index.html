常见shell需求总结

这两天把自己的一个Python项目用shell改写, 这里记录项目中遇到的一些基本需求

1. shell 函数

function func_name()
{
    func_body
}

func_name  # 调用

function 关键字可以省略
函数接受参数 

function hello()
{
    echo "Hello $1"
}

hello "neo1218"  # 调用

$0 代表函数本身, $1-9 依顺序代表参数, 10个以上需要 ${10} 调用。
函数的返回值只能是0~255
常用函数
1. log 函数 

function log()
{
    printf " \033[36m%10s\033[0m: \033[90m%s\033[0m\n" $1 $2
}

这里和Linux终端特殊颜色字符使用, 使log更"好看"
2. error_exit 函数 

function error_exit()
{
    echo $ 2>&1
    exit 1
}

exit 表示退出状态, exit 0 表示成功退出, 非0表示异常退出.
这里还有一个很有意思的地方: 2>&1
这里的1是stdout , 2是stderr, 2>&1 就是将标准错误重定向到标准输出, 用于错误处理。
3. help 系列函数
即显示信息的函数

function display_msg()
{
    cat <<-EOF
    msg msg msg
    msg msg msg
EOF
}

这样就可以很好的在终端显示信息啦
4. 其他
shell函数还有一个很有意思的特性, 关于shell的作用域, 即函数中可以直接对全局变量进行修改, 这样就可以很方便的在不同函数中调用全局状态, 对于简单的面向过程程序是非常友好的。
当然也可以在函数中定义局部变量local lvar

2. shell数组

shell支持数组的使用, 但是索引是从1开始...
1. 定义数组
array=()
2. 取数组元素值
array[4]
3. 获取数组的长度
${#array[@]}
4. 获取数组最后一个元素
${array[${#array[@]}]}

3. shell 判断

shell支持一系列的判断
1. 判断路径是否存在
if test -d "$path"
2. 判断字符串是否为空
if test -z $string
3. 判断是否存在参数
if test $# -eq 0

4. case 语句

shell 支持case语句 

case $var in
    xxx)  do sth ;;
    xx|xxx) do sth ;;
esac

5. 处理用户参数

由于c是命令, 所以处理用户参数就是必须的, 这里给个总结模版

if test $# -eq 0; then
    do sth
else
    while test $# -ne 0; do
        case $args in
            case1)
                do sth ;;
            -h|--help)
                display_help ;;
            -v|--version)
                show_version ;;
        esac
        shift
    done
fi

6. 其他

  1. $? 用于获取最近的返回值
  2. 赋值变量与被赋值变量中间的等号不含空格

总结

从python到shell, 是想挑战一下, 而且shell的速度快, 直接shell执行。像这种命令式面向过程的程序是很适合用shell编写的。
